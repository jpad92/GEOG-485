Question 3

The hits gave very useful information in regarding how to fulfil the GetCapabilities requests. However, it is not as common for the hits to give actual live WMS services as opposed to examples. When writing the example of the web address, they would use "servername" or "hostname" for the URL domain name.

Question 4 (Service #1)

  • http://LIDARserver.com/vancouver?SERVICE=WMS&REQUEST=GetCapabilities
  • Lidar WMS as powered by Lidar Server from GCoherent Software
  • The formats that are available for the GetMap requests from the service are: image/png, image/gif, and image/jpeg.
  • There are four layers included in the service.

Question 4 (Service #2)

  • http://suite.opengeo.org/geoserver/wms?service=WMS&version=1.3.0&request=GetCapabilities
  • GeoServer Web Map Service
  • The formats that are available for the GetMap requests from the service are: image/png, application/atom+xml, application/pdf, and application/vnd.google-earth.kml+xml. Then there is a format tab of application/vnd.google-earth.kml+xml;mode=networklink with the formats of application/vnd.google-earth.kmz, image/geotiff, image/geotiff8, image/gif, image/jpeg, image/png, image/svg+xml, image/tiff, and image/tiff8 under this tab.
  • There are 23 layers included in the service.
