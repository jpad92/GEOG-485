The "Get Capabilities" request type is common across all three OGC web services.

Question 2

  • The required parameters are the requests of the service that are required to add in order to fufill a full WMS request. The required parameters for a WMS request are: WMTVER=1.0.0(request the version type), SERVICE=WMS (service type), and REQUEST=GetCapabilities (request name).
  • With a GetCapabilitiese request, information on the service itself is given such as the title, name, abstract, and contact information of the service. Then, the actual capabilities of the file is given with the format of the dataset. Map information of the dataset is given in the GetMap section of the request such as the possible formats of the map and feature information of different possible layers within the map.

Question 3

  • The required parameters for a WMS GetMap request are: VERSION=version (need to which version you are writing in), REQUEST=GetMap (request name), LAYERS=layer_list (need to name and pinpoint which map layers wanted), STYLES=style_list (rendering style per requested layer), SRS(or CRS)=namespace:identifier (define spatial reference system of map), BBOX=minx,miny,maxx,maxy (define the bounding box corners of map area), WIDTH=output_width & HEIGHT=output_height (width & height in pixels of map picture), and FORMAT=output_format (the output format wanted for map).
  • A picture of the map you defined in the GetMap request is returned in response to a WMS GetMap request.
  • The significance of transparancy is whether you want to map background should to be transparent or not. The more transparent, the more the map background stands out.
